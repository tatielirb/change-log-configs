# Change Log Configs

Projeto aplicacao de organizacao de Pull Resquest em tag-release.

- Uso de pattern [git-commit-msg-linter](https://github.com/legend80s/git-commit-msg-linter#readme) para padronizar as mensagens de commit
- Uso de pattern [PRLint Reloaded](https://github.com/maor-rozenfeld/prlint-reloaded) com app [github](https://github.com/apps/prlint-reloaded) com base no template do [prlint](https://github.com/ewolfe/prlint) 
- Uso de change log paa criar lista de PR em tag release, com action [release-action](https://github.com/ncipollo/release-action), auxilio do [semantic-release](https://github.com/semantic-release/semantic-release) e [release-changelog-builder-action](https://github.com/mikepenz/release-changelog-builder-action) 
- Utilização de [Semantic Versioning](https://semver.org/) para o padrão de criação de Tag releas.

## Setup

Make sure to install dependencies:

```bash
# npm
npm install

```

## Development Server

Start the development server on `http://localhost:3000`:

```bash
# npm
npm run dev

```

## Production

Build the application for production:

```bash
# npm
npm run build

```

Locally preview production build:

```bash
# npm
npm run preview

```
